• ユニットモジュールのコンテナは、必ずdiv要素で、水平貫通の矩形ブロックとしてレイアウトされます。
  • ユニットモジュールの上下には、標準的なマージンが空くように設計されます。
  • ユニットモジュールは、特に記載のない限り、他のユニットモジュールやパーツモジュールと組み合わせて使うことはできません。
  • ユニットモジュールの名称は、プレフィックス unit_ から始まります(汎用ユニット .unit のみ例外)。コンテンツが独自にユニットモジュールを定義する場合のプレフィックスは、cont_unit_ です。

汎用ユニット

div.unit はプレーンなユニットです。

<div class="unit">
  <p><code>div.unit</code> はタダのユニットです。</p>
</div><!-- /.unit -->

div.unit で囲われた領域は、プレーンなユニットとして定義され、シンプルな矩形エリアとして扱われます。
div.unit 内には、任意の複数のパーツモジュールを組み込んで使用することができます。
div.unit の上下には標準的なマージンが設定され、最初の子要素の margin-top および最後の子要素の margin-bottom は 0px にセットされます。

div.unit の拡張

div.unit が提供する機能は、ユニットモジュールの最低限のスタイル設定(上下のマージン、矩形維持)のみです。この性質を利用するために、新しいカスタムユニットは div.unit を拡張して実装することができます。
下記はその例です。

<style type="text/css">
/* .cont_unit_sample */
#content .cont_unit_sample{
}
/* / .cont_unit_sample */
</style>

<div class="unit cont_unit_sample">
	<p>カスタムユニットの実装例</p>
</div>

この例は、div.unit と、カスタムユニットモジュールの .cont_unit_sample の両方をクラスに指定しています。.cont_unit_sample は、div.unit の性質を引き継ぎます。

ペイン(カラム)分割ユニット

ペイン分割ユニットは、複数のペインにユニットを均等分割します。
ただし、handheldなどの画面が小さいデバイスに対しては、通常通り縦のリキッドレイアウトが提供されます。

分割されたそれぞれのペイン(例えば .pane2L, .pane2R, .pane3L, .pane4L2, .pane5L2 など)内には、任意の複数のパーツモジュールを組み込んで配置することができます。

ページャーユニット

mk_pager( 1000 , 1 , 5 , array( 'href'=>'javascript:;' ) ); ?>

ページャーは、$theme->mk_pager() を通じて作成します。Pickles Framework 0.6.3 以降で利用可能です。

<?php
  print $theme->mk_pager( ${総件数} , ${現在のページ番号} , ${ページ当たりの表示件数} );
?>

$theme->mk_pager() が出力するHTMLソースは、次のような構造です。

mk_pager( 20 , 1 , 5 , array( 'href'=>'#' ) ) );
?>
  • ※$theme->mk_pager() の使い方について、詳しくはクラスリファレンスページを参照してください。
